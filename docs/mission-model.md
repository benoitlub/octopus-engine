# Mission Model

Une mission est l'objectif à réaliser.

Elle ne doit pas être confondue avec le workflow.

## Mission

La mission décrit le résultat attendu.

Exemple : trouver 20 prospects à contacter.

## Workflow

Le workflow décrit la manière d'y parvenir.

Une même mission peut avoir plusieurs workflows selon les policies.

## Conductor

Le Conductor comprend la demande humaine et formule une mission claire.

## Coordinator

Le Coordinator exécute le workflow choisi pour cette mission.

## Pourquoi cette distinction compte

Si mission et workflow sont confondus, chaque nouvelle stratégie devient une nouvelle logique métier.

Si elles sont séparées, Octopus peut adapter la manière d'agir sans changer l'intention de départ.
